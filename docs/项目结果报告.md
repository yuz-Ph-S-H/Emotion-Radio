# 项目结果报告

## 项目名称

「共感电台」——文字驱动的实时情绪音乐与视觉生成系统

## 完成情况

已根据 `项目实现文档.md` 完整生成项目文件，源码内容与文档中对应代码块保持一致。项目采用 Flask + Flask-SocketIO 作为后端，使用 jieba、SnowNLP 与意象词典完成本地情绪分析，并通过浏览器端 Tone.js、Canvas 粒子系统和 WebSocket 实现文字输入后的音乐与视觉联动。

## 已生成目录与文件

- `app.py`：Flask 主应用与 Socket.IO 事件入口。
- `config.py`：NLP 模式、API 参数和服务启动配置。
- `requirements.txt`：后端依赖清单。
- `nlp/`：本地分析器、API 分析器、情绪参数模型、意象词典和包初始化文件。
- `templates/index.html`：主页面结构。
- `static/css/style.css`：页面与交互样式。
- `static/js/`：工具函数、WebSocket、粒子系统、视觉引擎、音乐引擎、时间线和主控逻辑。
- `data/custom_dict.txt`：预留 jieba 自定义词典文件。
- `提示词示例.md`：按情绪、视觉效果和演示流程整理的可复制输入示例。
- `数学推导说明.md`：用公式整理情绪变量、意象权重、音乐参数和视觉参数的数学映射关系。
- `API权重与外部音乐说明.md`：说明外部轻音乐资产替换方式与 API 权重生成调用方法。
- `static/audio/`：内置三条本地原创轻音乐背景床和播放列表。
- `tools/generate_audio_assets.py`：用于重新生成原创轻音乐 WAV 资产。

.\venv\Scripts\python.exe -m pip install --no-cache-dir -r requirements.txt

## 混合场景修复

已修复混合场景只播放单一音色的问题。后端现在会在全文中扫描意象词，避免“下着雨”等短语漏掉“雨”；音乐参数也会返回 `instruments` 多音色列表。前端音乐引擎会将多个音色分配到主旋律、和声、点缀声部和鼓点，使“雨 + 彩虹 + 阳光 + 花园 + 雷声”这类复合输入产生多层音频反馈。

## 视听优化

已对视觉与音频进行二次优化。视觉端增加漂移渐变、补色光、环境色幕、轻微颗粒感、粒子淡入淡出与深度变化，并降低混合场景粒子堆叠强度。音频端保持实时生成，不使用外部热门片段或固定采样；生成逻辑改为更慢、更稀疏、更柔和的轻音乐结构，使用长释放音色、轻微延迟、混响和多声部点缀组合。

## 外部轻音乐与权重 API

已加入外部轻音乐背景床机制，当前使用项目本地生成的原创 WAV 文件，前端会根据 `music.audio_bed` 低音量循环播放，并叠加实时生成声部。API 模式已扩展 `imagery_weights` 返回结构，并提供 `generate_weights` 方法及 Socket.IO 事件，用于单独生成混合场景中各意象权重。

## 情绪贴合与听感优化（v2）

针对"音乐粗糙难听"与"映射跳变"问题，做了系统性优化，并同步到 `数学推导说明.md`（§16~§22）：

- **连续情绪参数**：后端 `compute_music_params` 新增 `brightness`（情绪明度）、`filter_cutoff`（音色低通截止，按 2 的指数感知映射）、`seventh`/`ninth`（和弦七/九度增色）、`contour`（旋律走向偏置）、`step_size`（步幅）、`attack`（音头）。情绪连续变化时音色、和弦、旋律走向随之平滑变化，不再只靠离散阈值跳变。
- **母带处理链**：前端 `MusicEngine` 增加"低通 → EQ → 压缩 → 限幅"母带链，低通截止由情绪明度实时驱动，限幅器钳制峰值在 −2 dB，彻底消除爆音；各合成器音量与鼓点音量下调，听感更柔。
- **旋律优化**：旋律按 `contour` 偏置走向（积极上行/消极下行）、按 `step_size` 控制跳进，并在音域边缘温和回中，避免越走越偏。和弦按情绪加入大七/小七/九度增色。
- **背景床重制**：`tools/generate_audio_assets.py` 改为暖色加性合成（谐波滚降）+ 升余弦音头（消除逐拍爆音）+ 微失谐合唱 + 一阶低通 + Schroeder 混响 + 无缝循环交叉淡化 + 立体声（32 kHz）。三条床 `calm_piano` / `warm_guitar` / `night_pad` 已重新生成。
- **视觉美化**：网页改为玻璃拟态/电影感模板（顶部品牌条、情绪解析玻璃卡片、渐变流光标题、发光输入条、状态指示器），并引入 Web 字体；粒子端为星光/萤火/微光加入微闪烁、柔化外发光渐变。

以上改动均通过本地分析、Socket.IO 端到端（`emotion_result` 含全部新字段）、首页与静态资源 HTTP 200 验证。

## 场景区分度与乐句优化（v2.1）

针对"场景听感趋同、旋律简单反复、粒子杂乱"做了第三轮优化（推导见 `数学推导说明.md` §23~§24）：

- **情绪性格档**：后端按 $(v,a)$ 与意象把场景判为 6 种性格（忧伤/平静/温柔/欢快/紧张/梦幻），每种性格对应不同的旋律句式、声部上限、是否用鼓、是否点缀、速度与力度系数。各场景在配器与节奏上同时拉开，区分度显著提升。
- **选择性配器**：乐器按性格上限取前 2~3 个，不再"所有场景都叠满全部乐器"；鼓点只在紧张/欢快场景出现，点缀只在温柔/欢快/梦幻出现。
- **乐句引擎**：前端旋律改为以"乐句"为单位生成——从对应性格的乐句库取句、做倒影变形、按情绪定起始锚点、句内力度递减形成呼吸、句末留白换气，并沿 Transport 时间轴按变时值精确排程，彻底取代逐拍随机游走的"简单反复"。
- **关键词扩充**：情绪关键词由 93 个增至 197 个（积极 90 / 消极 107），覆盖更多情感细分，效价判定更稳更细。
- **粒子净化**：同屏粒子效果上限收为 2（主效果原速、次效果 45%），并下调粒子数量、速度、辉光与颗粒噪点，存活粒子上限降至 480，画面更干净、主次分明。

实测六种性格输出（句式/鼓点/声部/速度/粒子）均明显不同，且通过 Socket.IO 端到端验证返回 `mood`、`phrase_style`、`use_drums`、`accent`、`max_voices` 等新字段。

## 运行方式

当前开发服务器已启动，访问地址：

```text
http://127.0.0.1:5000
```

如需手动重新启动，建议在 Windows PowerShell 中使用 UTF-8 输出模式：

```powershell
.\venv\Scripts\python.exe -X utf8 app.py
```

## 注意事项

项目默认使用本地 NLP 方案，不需要 API 密钥。若切换到 API 增强方案，需要在 `config.py` 中设置 `NLP_MODE = "api"` 并填写有效服务商密钥。Windows 环境下 eventlet 与 jieba 初始化期间可能输出兼容性告警，但已验证首页加载和 Socket.IO 分析事件可正常工作。
