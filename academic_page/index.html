系统总览。一段中文文本经情感分析得到连续的情绪变量 效价 $v$、唤醒度 $a$ 与意象权重 $W$，再被同步映射为浏览器端实时合成的 多声部轻音乐与粒子视觉。系统不把文本归为单一情绪标签，而是将其解释为 多意象、多权重、多声部、多粒子的连续跨模态映射。

摘要 Abstract

共感电台是一套文字驱动的实时情绪音乐与视觉生成系统。 用户输入任意中文文本，系统通过 jieba 分词、SnowNLP 基础情感判定、自建意象词典与情绪关键词的 三级情感融合，估计出连续的情绪效价 $v$ 与唤醒度 $a$，并对句中多个意象计算归一化权重。 这些情绪变量被进一步映射为一组连续的音乐与视觉参数—— 调式、速度、力度、音色低通截止频率、和弦色彩、旋律走向，以及背景配色、粒子效果与运动强度。

后端基于 Flask + Flask-SocketIO，前端使用 Tone.js 实时合成音频、Canvas 粒子系统渲染视觉， 通过 WebSocket 实现"输入即响应"的低延迟联动。为了避免离散阈值导致的"音乐跳变"与早期实现的 "粗糙听感"，系统引入情绪明度统一驱动量、母带处理链（低通→EQ→压缩→限幅）、 以及以乐句为单位、按六种情绪性格选择句式与配器的算法作曲引擎。 最终输出在保持情感贴合的同时具备连续、温暖、耐听的轻音乐质感。本页系统性地呈现其方法、数学映射与演示结果。

方法 Method

1 三级情感融合：从文本到 $(v,a)$

SnowNLP 对中文短句的判定整体偏积极（大量中性/略消极句被判到 $0.8\sim0.95$），若直接采用会让"暴雨""深夜" 也被映射为明亮大调。为此系统采用三级结构：SnowNLP 给出整体语气，意象效价先验给出场景情感基调， 情绪关键词给出强烈的局部修正。

意象效价先验（按意象权重 $w_k$ 加权放大，$G=3.5$）：

$$\bar\delta=\frac{\sum_{k}w_k\,\delta_k}{\sum_{k}w_k},\qquad v_{\mathrm{img}}=\operatorname{clip}\bigl(0.5+G\,\bar\delta\bigr)$$

与 SnowNLP 基础效价 $v_0$ 融合（以意象为主，$\lambda=0.4$），再叠加情绪关键词修正 $d_j$：

$$v_b=\operatorname{clip}\bigl(\lambda v_0+(1-\lambda)v_{\mathrm{img}}\bigr),\qquad v=\operatorname{clip}\Bigl(v_b+\textstyle\sum_j d_j\Bigr)$$

唤醒度 $a$ 由默认值出发，结合标点（感叹号 $+0.08$、问号 $+0.05$、省略号 $-0.05$）、意象加权唤醒度 $a_I=\sum_k w_k a_k$ 与短文本补偿 $b$ 共同决定： $a=\operatorname{clip}(0.4a_p+0.6a_I+b)$。经此改造， "暴雨狂风" $\to v\approx0.38$（小调）、 "阳光花园" $\to v\approx1.0$（大调），音乐显著更贴合情感。

2 意象权重：多意象的占比分配

一句话里可能出现多个意象。系统为第 $k$ 个意象按先验唤醒度与出现顺序打分并归一化，使权重之和为 $1$， 权重进而影响唤醒度、音色排序与视觉效果优先级：

$$s_k=a_k+0.02\cdot\max(0,\,12-p_k),\qquad w_k=\frac{s_k}{\sum_j s_j}$$

示例 "雨停以后出现彩虹，阳光照进花园" → $w_{雨}=0.242,\; w_{彩虹}=0.311,\; w_{阳光}=0.265,\; w_{花园}=0.182$。

3 情绪性格档：六种性格拉开场景区分度

为避免"所有场景都叠满全部乐器、听感趋同"，系统先由 $(v,a)$ 与意象集合把场景判为六种情绪性格， 每种性格对应不同的旋律句式、声部上限、是否用鼓与点缀，以及速度/力度系数 $k_t,k_v$。

4 连续情绪参数：让音色与和弦随情绪平滑变化

为消除"离散阈值跳变"，系统定义情绪明度 $\beta$ 作为统一驱动量， 再以人耳近似对数感知的指数映射驱动音色低通截止频率 $f_c$：

$$\beta=\operatorname{clip}(0.18+0.62v+0.20a),\qquad f_c(\beta)=500\cdot 2^{\,1+3\beta}\ \text{Hz}$$

$\beta=0 \Rightarrow f_c=1000\,$Hz（暖、暗、克制）；$\beta=1 \Rightarrow f_c=8000\,$Hz（通透、明亮）。

调式由效价分段（$v>0.65$ 大调五声、$v\le0.30$ 小调），并按情绪加入七度/九度增色使色彩更开阔； 旋律走向偏置 $\kappa=\operatorname{clip}(2(v-0.5),-1,1)$ 决定每个乐句的起始锚点——积极情绪整段偏高、消极偏低。

调式 / 根音

由效价 $v$ 分四段，大/小调与五声音阶平滑切换。

音色亮度

低通截止 $f_c$ 由情绪明度实时 ramp，暗↔亮连续过渡。

和弦色彩

按情绪加入大七/小七、九度增色，强消极保持朴素三和弦。

旋律走向

走向偏置 $\kappa$ 决定乐句锚点，整体上行/下行随情感同向。

5 乐句引擎与母带处理链：从"反复音型"到耐听轻音乐

旋律不再逐拍随机游走，而是以乐句为单位生成：从当前性格的乐句库取句 → 以 $0.35$ 概率做倒影变形 → 按 $\kappa$ 定起始锚点 → 句内力度递减形成"呼吸" → 句末留白换气，并沿 Transport 时间轴按变时值精确排程。 所有声部与背景床最终汇入统一母带链：

声部 Delay / Reverb 低通 $f_c$ 三段 EQ 压缩 限幅 −2 dB

限幅器把峰值钳制在 $-2\,$dB 以下，彻底杜绝爆音；背景音乐床为本地原创加性合成 （谐波滚降 + 升余弦音头 + 微失谐合唱 + 低通 + Schroeder 混响 + 无缝循环交叉淡化），无版权问题。

6 视觉映射：情绪到色彩与粒子

背景使用 HSL 色彩空间，色相由效价控制（高效价偏暖、中性偏紫蓝、低效价偏深蓝），饱和度与亮度随唤醒度提高。 粒子效果按意象权重降序只取前 2 个（主效果原速、次效果 $0.45$ 倍速）以保持画面主次分明； 同屏存活粒子上限 $480$，速度、数量、辉光均随唤醒度线性映射：

$$\text{speed}=0.28+0.92a,\quad \text{count}=\lfloor 20+80a\rfloor,\quad \text{glow}=0.2+0.5a$$

关键对比：同样的"雨"，不同的情绪

系统并非按关键词查表，而是综合情感效价生成音乐——两句都含意象"雨"，音乐走向却截然相反。

$v = 0.38$ · 小调 · 暗

"窗外是冰冷的小雨，房间里只剩回忆和叹息。"

同样有"雨"，情绪低落 → 忧伤，旋律下行、暗色粒子。

$v = 0.86$ · 大调 · 亮

"雨停以后出现彩虹，阳光照进花园，远处还有雷声。"

同样有"雨"，但整体明亮 → 温暖，多声部 + 多粒子叠加。

体现意象效价先验 + 三级情感融合：复合情绪 "雨/彩虹/阳光/花园/雷声" 被分配为 鼓点 + 钟琴 + 吉他 + 钢琴 四声部，粒子 闪电/彩虹/阳光/雨/花瓣 同时出现。

技术栈与运行 Stack & Run

后端

  • Flask + Flask-SocketIO（WebSocket 实时通信）
  • jieba 分词 · SnowNLP 情感判定
  • 自建意象词典 + 情绪关键词（197 个）
  • 可选 API 增强方案（兼容 OpenAI 接口）

前端

  • Tone.js 实时音频合成 + 母带链
  • Canvas 粒子系统（视觉引擎）
  • 乐句引擎 / 时间线调度
  • 玻璃拟态电影感 UI
